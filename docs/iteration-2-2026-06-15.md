# 第二轮迭代记录

日期：2026-06-15

## 目标

将第一轮的可用版本推进到可持续维护版本，解决配置漂移、缺少浏览器端测试、
重复托盘实例和无 CI 的问题。

## 已完成

- `config/tts_catalog.json` 成为声音、语速和默认值的唯一来源。
- 服务端、托盘和内置测试页运行时读取 catalog。
- 用户脚本中的 catalog 由 `scripts/sync_catalog.py` 生成并在 CI 中检查。
- 用户脚本核心可以在 Node 中导入，自动测试请求代际取消和 Blob URL 清理。
- Windows 命名互斥阻止重复托盘实例。
- 托盘区分自有服务与外部服务，外部服务不显示 Stop/Restart。
- `requirements-test.txt` 允许 CI 在不安装 Torch/Kokoro、不下载模型的情况下运行。
- GitHub Actions 覆盖 Python/JavaScript 语法、单元测试、catalog 同步和依赖检查。
- 用户脚本增加 GitHub Raw 更新元数据。

## 测试策略

- Python：API、音频处理、catalog、无 Torch 导入、Windows mutex、托盘归属。
- JavaScript：无浏览器导入、旧请求取消、过期回调隔离、Blob URL 幂等释放。
- 真实模型：发布前在本机 CUDA 环境执行美式和英式短句烟雾测试。

## 验证结果

- Python 单元测试：19 项通过。
- JavaScript 单元测试：4 项通过。
- Windows 隔离 CI 环境：语法、测试、catalog 同步和依赖检查全部通过。
- CUDA 烟雾测试：`af_bella` 与 `bf_emma` 均返回有效 PCM WAV。
- 运行服务：版本 `1.1.0`，设备 `cuda`。
